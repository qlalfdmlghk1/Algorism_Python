현재 열려있는 파일을 읽고 코딩 문제 풀이를 리뷰해줘.

## 파일명 규칙

- `BOJ_#####` → 백준 온라인 저지 #####번 문제
- `PRO_#####` → 프로그래머스 #####번 문제

파일명에서 문제 번호를 파악하고, 해당 문제의 내용을 기반으로 리뷰해줘.

## 리뷰 모드

사용자의 추가 입력: $ARGUMENTS

- `$ARGUMENTS`가 비어있으면 → **맞았을 때** 리뷰
- "틀렸어", "틀림", "오답" 등이 포함되면 → **틀렸을 때** 리뷰
- "모르겠어", "못풀겠어", "접근" 등이 포함되면 → **접근을 못했을 때** 리뷰

---

### 1. 맞았을 때 리뷰

다음 항목들을 포함해서 리뷰해줘:

1. **문제 분류** - 이 문제의 유형 (DP, 그리디, 그래프, 이분탐색 등)
2. **접근법 분석** - 사용자가 어떤 알고리즘/전략을 사용했는지 정리
3. **시간/공간 복잡도** - 현재 코드의 복잡도 분석
4. **잘한 점** - 코드에서 잘 작성된 부분에 대한 긍정적 피드백
5. **개선 제안** - 더 효율적이거나 깔끔한 방법이 있다면 제안 (없으면 생략)
6. **다른 풀이법** - 의미 있는 대안적 접근이 있다면 소개 (없으면 생략)
7. **오답노트** - 이 문제에서 배워야 할 핵심을 3줄 분량으로 정리해줘. 나중에 복습할 때 바로 떠올릴 수 있도록 간결하게.
8. **관련 문제 추천** - 비슷한 유형으로 연습할 문제 추천

### 2. 틀렸을 때 리뷰

다음 항목들을 포함해서 리뷰해줘:

1. **문제 분류** - 이 문제의 유형
2. **어디서 틀렸는지** - 코드의 버그, 논리 오류, 놓친 엣지 케이스 등을 구체적으로 지적
3. **왜 틀렸는지** - 원인 분석 (잘못된 가정, 알고리즘 선택 오류, 구현 실수 등)
4. **힌트 / 수정 방향** - 직접 정답을 알려주기보다, 어떤 방향으로 수정해야 하는지 가이드
5. **오답노트** - 이 문제에서 배워야 할 핵심을 3줄 분량으로 정리해줘. 나중에 복습할 때 바로 떠올릴 수 있도록 간결하게.

### 3. 접근을 못했을 때 리뷰

다음 항목들을 포함해서 리뷰해줘:

1. **문제 분류** - 이 문제의 유형
2. **핵심 관찰 포인트** - 문제에서 캐치했어야 할 핵심 조건이나 힌트
3. **사고 유도** - "문제의 ~~~ 조건을 보고 ### 알고리즘을 떠올렸어야 한다" 식으로 문제 해석 → 알고리즘 연결 과정을 설명
4. **풀이 방향 가이드** - 전체 정답 코드가 아니라, 어떤 순서로 접근하면 좋을지 방향성 제시
5. **오답노트** - 이 문제에서 배워야 할 핵심을 3줄 분량으로 정리해줘. 나중에 복습할 때 바로 떠올릴 수 있도록 간결하게.
